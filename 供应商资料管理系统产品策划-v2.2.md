# 供应商资料管理系统产品策划 v2.2

## 修正核心理念

**基于SQE工程师实际工作场景的精准解决方案**

> "错误！！就是要分别管理，而且要可以自定义物料（新增父级文件）"
> "不是通过邮件预警，而是实现邮件自动通知供应商提供新的报告"

## 一、用户场景分析

### 1.1 真实用户场景
**角色**: SQE工程师（供应商质量工程师）
**工作内容**: 管理胶壳供应商的各类认证文件(这只是例子，可能是其他物料的供应商)
**具体挑战**:
- 胶壳供应商有电木粉的ROHS、REACH、HF证书（3个文件）
- 同一供应商还有PIN脚的ROHS、REACH、HF证书（另外3个文件）
- **关键**: 6个文件的到期日期可能完全不同
- **需求**: 分别管理每个文件，但要有层级关系便于查看

### 1.2 用户痛点
1. **文件到期时间不一致**，无法批量更新
2. **需要通知供应商提供新报告**，而非内部预警
3. **物料结构自定义**，不同供应商物料组成不同
4. **操作要简单直接**，避免复杂的自动识别

## 二、核心功能设计

### 2.1 自定义物料层级管理 ⭐核心功能

#### 2.1.1 物料树形结构
```
供应商A
├── 胶壳（胶壳的组成部分为电木粉和pin脚，如果胶壳有整体的检测证书，则不需要细分成分）
│   ├── 电木粉
│   │   ├── ROHS证书 (2025-06-30到期)
│   │   ├── REACH证书 (2025-12-31到期)
│   │   └── HF证书 (2025-09-15到期)
│   └── PIN脚
│       ├── ROHS证书 (2025-08-20到期)
│       ├── REACH证书 (2025-11-30到期)
│       └── HF证书 (2025-07-10到期)
└── 其他物料
    └── (可自定义添加)
```

#### 2.1.2 自定义物料功能
- **新增父级物料**: 用户可创建"胶壳组合"、"端子组合"等自定义分类
- **新增子级物料**: 在父级下添加"电木粉"、"PIN脚"等具体物料
- **灵活配置**: 每个物料可有独立的认证要求
- **模板复制**: 可将常用物料结构复制到其他供应商

### 2.2 文件分别管理系统

#### 2.2.1 独立文件管理
- **每个文件独立**: 认证文件完全独立管理（数量不固定）
- **独立到期时间**: 每个文件有自己的到期提醒
- **独立状态跟踪**: 每个文件的状态单独显示

#### 2.2.2 层级化展示
- **树形视图**: 按物料层级显示所有文件（不一定是树形，你额可以有更好的展现方式）
- **状态汇总**: 父级显示子级文件的整体状态
- **快速定位**: 点击层级快速定位到具体文件

### 2.3 邮件自动通知系统 ⭐关键功能

#### 2.3.1 邮件模板预编辑
```
邮件模板编辑器
├── 识别什么证书到期（统一模板，通过变量区分）
│   ├── ROHS到期
│   ├── REACH到期
│   └── HF到期 （当然还有MSDS,质量保证协议等）
├── 变量设置
│   ├── {供应商名称}
│   ├── {物料名称}
│   ├── {证书类型}
│   ├── {到期日期}
│   └── {SQE工程师联系方式}
└── 预览功能
    └── 实时预览邮件内容
```

#### 2.3.2 自动通知机制
- **到期前30天**: 自动发送第一次提醒
- **到期前15天**: 自动发送第二次提醒
- **到期前7天**: 自动发送紧急提醒
- **过期当天**: 自动发送过期通知


### 2.4 表格直接上传功能 ⭐效率提升

#### 2.4.1 表格内嵌上传
```
供应商资料管理表格
├── 供应商A
│   ├── [+] 胶壳组合
│   │   ├── 电木粉
│   │   │   ├── ROHS: [上传文件] 2025-06-30 [编辑]
│   │   │   ├── REACH: [上传文件] 2025-12-31 [编辑]
│   │   │   └── HF: [上传文件] 2025-09-15 [编辑]
│   │   └── PIN脚
│   │       ├── ROHS: [上传文件] 2025-08-20 [编辑]
│   │       ├── REACH: [上传文件] 2025-11-30 [编辑]
│   │       └── HF: [上传文件] 2025-07-10 [编辑]
│   └── [+] 添加物料组合
└── 供应商B
```

#### 2.4.2 上传体验优化
- **拖拽上传**: 直接拖拽文件到对应位置
- **批量选择**: 可选择多个文件一次性上传到不同位置
- **进度显示**: 实时显示上传进度
- **即时预览**: 上传后立即显示文件信息

#### 2.4.3 本地文件管理架构
```
uploads/
├── 供应商A/
│   ├── 电木粉/
│   │   ├── ROHS报告/
│   │   ├── REACH报告/
│   │   ├── MSDS报告/
│   │   └── HF报告/
│   └── PIN脚/
│       ├── ROHS报告/
│       ├── REACH报告/
│       ├── MSDS报告/
│       └── HF报告/
├── 供应商B/
│   └── (按相同结构)
└── ...
```
- **自动同步**: 系统新增、删除文件时自动同步到本地文件夹
- **层级对应**: 文件夹结构与系统中的物料层级完全对应
- **版本管理**: 同一类型证书的新版本自动归档到对应文件夹

## 四、技术架构优化

### 4.1 前端技术栈
- **组件化设计**: 物料管理组件化，便于复用
- **状态管理**: 使用Redux管理复杂状态
- **虚拟滚动**: 处理大量数据时的性能优化
- **离线支持**: 支持离线查看和操作

### 4.2 后端技术栈
- **微服务架构**: 物料管理、邮件服务、文件服务分离
- **消息队列**: 处理邮件发送等异步任务
- **缓存优化**: Redis缓存热点数据
- **全文搜索**: Elasticsearch支持快速搜索

### 4.3 数据库设计
- **树形结构存储**: 使用Closure Table存储物料层级
- **版本管理**: 文件版本控制
- **审计日志**: 完整的操作日志记录

## 五、本地文件管理架构设计

### 5.1 文件夹结构规范
```
uploads/
├── {供应商名称}/
│   ├── {物料名称}/
│   │   ├── {证书类型}/
│   │   │   ├── 文件1.pdf
│   │   │   ├── 文件2.pdf
│   │   │   └── ...
│   │   └── ...
│   └── ...
└── ...
```

### 5.2 文件同步机制
- **上传同步**: 系统上传文件时自动创建对应文件夹结构
- **删除同步**: 系统删除文件记录时同步删除本地文件
- **更新同步**: 文件更新时保留历史版本，最新版本放在对应文件夹
- **结构同步**: 物料结构调整时自动调整文件夹结构

### 5.3 文件命名规范
- **格式**: `{供应商名称}_{物料名称}_{证书类型}_{版本号}_{日期}.{扩展名}`
- **示例**: `供应商A_电木粉_ROHS_v2_20251201.pdf`
- **版本控制**: 自动递增版本号，历史文件归档保存
