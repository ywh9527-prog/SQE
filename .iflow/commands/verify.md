# Verification Before Completion Command

## 命令名称
`/verify` 或 `/verification-before-completion`

## 描述
完成前验证。确保问题真正解决，所有功能正常工作，没有引入新问题。

## 触发条件
- 声称工作完成
- 修复了 bug
- 实现了功能
- 准备提交代码

## 验证流程

### 1. 问题验证
- [ ] 原始问题已解决
- [ ] 解决方案有效
- [ ] 没有副作用
- [ ] 边界情况已测试

### 2. 功能验证
- [ ] 所有功能正常工作
- [ ] 用户场景已测试
- [ ] 集成测试通过
- [ ] 回归测试通过

### 3. 代码质量验证
- [ ] 代码审查通过
- [ ] 没有编译警告
- [ ] 代码风格一致
- [ ] 注释充分

### 4. 性能验证
- [ ] 性能指标达标
- [ ] 没有性能退化
- [ ] 资源使用合理
- [ ] 负载测试通过

### 5. 安全验证
- [ ] 安全检查通过
- [ ] 没有安全漏洞
- [ ] 权限控制正确
- [ ] 数据保护到位

## 验证方法

### 自动化验证
- 运行测试套件
- 代码静态分析
- 性能基准测试
- 安全扫描

### 手动验证
- 功能测试
- 用户验收测试
- 视觉回归测试
- 兼容性测试

### 验证记录
- 测试结果
- 性能指标
- 问题清单
- 改进建议

## 验证原则
- **证据优先**：基于测试结果而非声明
- **全面覆盖**：验证所有相关方面
- **可重复**：验证过程可重复
- **文档化**：记录验证结果

## 技能调用
自动调用 `verification-before-completion` 技能

## 输出
- 验证报告
- 测试结果
- 问题清单
- 完成状态

## 示例
```
/verify
验证用户登录功能是否完成
```

## 注意事项
- 不要跳过验证步骤
- 基于证据而非声明
- 记录所有发现
- 确保没有引入新问题
- 在确认完成前进行彻底验证
