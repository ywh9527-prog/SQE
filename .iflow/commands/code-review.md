# Code Review Command

## 命令名称
`/code-review` 或 `/requesting-code-review`

## 描述
请求代码审查。在任务之间进行代码审查，确保代码质量和符合最佳实践。

## 触发条件
- 完成一个功能模块
- 准备提交代码
- 需要质量保证
- 团队协作开发

## 审查清单

### 功能性
- [ ] 代码实现了预期功能
- [ ] 所有测试通过
- [ ] 边界条件已处理
- [ ] 错误处理完善

### 代码质量
- [ ] 代码清晰易读
- [ ] 遵循项目编码规范
- [ ] 没有重复代码
- [ ] 命名有意义

### 性能
- [ ] 没有明显的性能问题
- [ ] 资源使用合理
- [ ] 没有内存泄漏
- [ ] 算法复杂度合适

### 安全性
- [ ] 输入验证完善
- [ ] 没有安全漏洞
- [ ] 敏感数据保护
- [ ] 权限检查正确

### 可维护性
- [ ] 代码有适当注释
- [ ] 模块化设计
- [ ] 易于扩展
- [ ] 文档完善

## 审查流程

### 1. 自我审查
- 开发者先进行自我审查
- 运行所有测试
- 检查代码风格

### 2. 同行审查
- 提交审查请求
- 提供上下文信息
- 说明变更原因

### 3. 反馈处理
- 接收反馈意见
- 讨论问题
- 修改代码

### 4. 最终批准
- 所有问题解决
- 测试通过
- 批准合并

## 审查原则
- **建设性**：提供建设性反馈
- **尊重**：尊重开发者的工作
- **及时**：及时响应审查请求
- **彻底**：仔细检查每个细节

## 技能调用
自动调用 `requesting-code-review` 技能

## 输出
- 代码审查报告
- 问题清单（按严重性分类）
- 改进建议
- 批准/拒绝决定

## 示例
```
/code-review
请审查用户登录功能的代码
```

## 注意事项
- 重点关注关键问题
- 提供具体的改进建议
- 解释问题的严重性
- 保持专业和尊重的态度
