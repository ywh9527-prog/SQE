# Writing Plans Command

## 命令名称
`/plan` 或 `/writing-plans`

## 描述
编写详细实施计划。将工作分解为可管理的小任务（2-5分钟每个），每个任务都有确切的文件路径、完整代码和验证步骤。

## 触发条件
当有：
- 已批准的设计文档
- 需要实现的功能规格
- 复杂的开发任务
- 需要团队协作的任务

## 计划结构

### 任务分解原则
- **小任务**：每个任务 2-5 分钟可完成
- **具体路径**：明确要修改的文件路径
- **完整代码**：提供完整的代码实现
- **验证步骤**：如何验证任务完成

### 计划内容
1. **任务列表**：编号的任务清单
2. **依赖关系**：任务之间的依赖
3. **文件路径**：每个任务涉及的文件
4. **代码示例**：关键代码片段
5. **验证方法**：测试和验证步骤
6. **时间估算**：每个任务的预估时间

## 计划格式

```markdown
# 实施计划：[功能名称]

## 概述
简要描述要实现的功能

## 任务列表

### 任务 1: [任务名称]
- **文件**: `path/to/file.js`
- **描述**: 任务描述
- **代码**: 完整代码
- **验证**: 如何验证
- **时间**: 2-3分钟

### 任务 2: [任务名称]
...

## 依赖关系
- 任务 2 依赖任务 1
- 任务 3 依赖任务 2

## 验证清单
- [ ] 所有测试通过
- [ ] 功能正常工作
- [ ] 代码审查通过
```

## 计划原则
- **YAGNI**：你不需要它（只实现必需功能）
- **DRY**：不要重复自己
- **TDD**：测试驱动开发
- **小步骤**：保持任务小而专注

## 技能调用
自动调用 `writing-plans` 技能

## 输出
- 详细的实施计划文档
- 任务清单
- 依赖关系图
- 验证清单

## 示例
```
/plan
为用户登录功能编写实施计划
```

## 注意事项
- 任务必须小而具体
- 提供完整的代码示例
- 明确验证方法
- 考虑依赖关系
- 估算合理的时间
