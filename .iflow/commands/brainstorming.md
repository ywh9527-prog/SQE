# Brainstorming Command

## 命令名称
`/brainstorming`

## 描述
创意头脑风暴，在开始任何创造性工作前使用。帮助将想法转化为完整的设计和规格。

## 触发条件
当用户想要：
- 设计新功能
- 创建新组件
- 添加新功能
- 修改行为
- 探索设计选项

## 工作流程

### 步骤 1: 理解想法
- 检查当前项目状态（文件、文档、最近提交）
- 一次问一个问题来细化想法
- 优先使用多选题，必要时使用开放式问题
- 每条消息只问一个问题
- 专注于理解：目的、约束、成功标准

### 步骤 2: 探索方法
- 提出 2-3 种不同方案及其权衡
- 以对话方式展示选项，提供推荐和理由
- 以推荐选项为主，解释原因

### 步骤 3: 展示设计
- 一旦理解要构建的内容，展示设计
- 将设计分成小节（200-300字）
- 每节后询问是否正确
- 覆盖：架构、组件、数据流、错误处理、测试
- 准备返回澄清

### 步骤 4: 文档化
- 将验证后的设计写入 `docs/plans/YYYY-MM-DD-<topic>-design.md`
- 使用清晰简洁的写作风格
- 将设计文档提交到 git

### 步骤 5: 准备实施（可选）
- 询问："准备好开始实施了吗？"
- 使用 using-git-worktrees 技能创建独立工作空间
- 使用 writing-plans 技能创建详细实施计划

## 关键原则
- **一次一个问题** - 不要用多个问题压倒用户
- **多选题优先** - 可能时更容易回答

## 技能调用
自动调用 `brainstorming` 技能

## 输出
- 设计文档（Markdown 格式）
- Git 提交记录

## 示例
```
/brainstorming
帮我设计一个用户登录功能
```
