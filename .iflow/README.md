# Superpowers 工作流使用指南

## 📋 概述

本项目已配置 Superpowers 技能库的 iFlow CLI 工作流，提供了一套完整的软件开发工作流程和最佳实践。

### 1. 启动 iFlow CLI

```bash
cd D:\AI\Claude-SQE-Data-Analysis-Assistant-refactored

```

### 2. 使用斜杠命令

在 iFlow CLI 中，你可以直接使用以下斜杠命令：

## 🎯 可用命令

### 设计与规划
- `/brainstorming` - 创意头脑风暴
  ```
  /brainstorming
  帮我设计一个用户登录功能
  ```

- `/plan` - 编写实施计划
  ```
  /plan
  为用户登录功能编写实施计划
  ```

### 开发执行
- `/tdd` - 测试驱动开发
  ```
  /tdd
  使用测试驱动开发实现用户登录
  ```

### 质量保证
- `/debug` - 系统化调试
  ```
  /debug
  帮我调试这个登录失败的问题
  ```

- `/code-review` - 请求代码审查
  ```
  /code-review
  请审查用户登录功能的代码
  ```

- `/verify` - 完成前验证
  ```
  /verify
  验证用户登录功能是否完成
  ```

## 📖 使用流程

### 标准开发流程

1. **需求澄清**
   ```
   /brainstorming
   帮我设计一个新功能
   ```

2. **编写计划**
   ```
   /plan
   编写详细的实施计划
   ```

3. **实施开发**
   ```
   /tdd
   使用测试驱动开发实现功能
   ```

4. **调试问题**
   ```
   /debug
   帮我调试这个问题
   ```

5. **代码审查**
   ```
   /code-review
   请审查代码
   ```

6. **验证完成**
   ```
   /verify
   验证功能是否完成
   ```

## 📚 命令详解

### /brainstorming
- **用途**：创意头脑风暴，设计新功能
- **触发**：在设计新功能前使用
- **输出**：设计文档

### /plan
- **用途**：编写详细实施计划
- **触发**：有明确需求后使用
- **输出**：任务清单和实施计划

### /tdd
- **用途**：测试驱动开发
- **触发**：实现功能或修复bug时使用
- **输出**：测试代码和实现代码

### /debug
- **用途**：系统化调试
- **触发**：遇到bug或问题时使用
- **输出**：问题分析和修复方案

### /code-review
- **用途**：请求代码审查
- **触发**：完成代码后使用
- **输出**：代码审查报告

### /verify
- **用途**：完成前验证
- **触发**：声称完成时使用
- **输出**：验证报告

## 💡 最佳实践

1. **按顺序使用**：遵循标准开发流程
2. **小步快跑**：保持任务小而具体
3. **测试先行**：始终先写测试
4. **持续验证**：每步都进行验证
5. **文档化**：记录所有决策和发现

## 🔧 配置文件

工作流配置位于 `.iflow/` 目录：

- `IFLOW.md` - 核心配置
- `commands/` - 斜杠命令定义
- `agents/` - 技能代理配置
- `tasks/` - 任务定义
- `templates/` - 模板文件
